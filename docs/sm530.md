# SM530 Support Notes

This branch adds first-pass SM530 support for the Nelsonic Game Watch titles in current MAME:

- `nsmb3` - Super Mario Bros. 3
- `nsmw` - Super Mario World
- `nstarfox` - Star Fox

The implementation follows MAME's SM530 device model rather than introducing a separate package format. The ROM generator assigns SM530 packages CPU ID `3`, pads the 2K program ROM to the existing fixed `0x1000` program area, and appends the 256-byte melody ROM at `0x1000`, the same place used by the SM511/SM512 melody path.

## Generator Changes

- The manifest and converter understand `sm530` as a CPU type.
- SM530 packages are treated as melody-capable and require a `maincpu:melody` ROM.
- The MAME extractor can parse `CONS(...)` metadata as well as `SYST(...)` metadata.
- The MAME input parser can split an 8-bit `IN.n` port into two four-bit package rows. Nelsonic's low K nibble is stored in `S0`; its high KE nibble is stored in `S1`.

## RTL Changes

- CPU ID `3` selects SM530 decode, reset vector `0x3c0`, wake vector `0x000`, and the SM530 three-clock instruction cadence.
- SM530 opcodes were added for `KETA`, `ATS`, `ATF`, `SDS`, `RDS`, `TG`, `SABL`, `INIS`, `PRE`, `TR`, `TRS`, `TL`, and the SM530 variants of `ADX`, `LB`, and `INCB`.
- RAM address masking now supports SM530 `SABM` and `SABL`, plus the LCD RAM mirrors at `0x40/0x60` and `0x50/0x70`.
- SM530 LCD RAM is exposed through the existing SM500-style segment normalizer: x is the output group, y is segment bit 0-3, and z selects H1/H2.
- The live SM530 LCD output is latched from the canonical LCD RAM ranges `0x40-0x4b` and `0x50-0x5b`; writes through the mirror ranges still land in those canonical addresses before video sees them.
- The divider now supplies SM530 gamma flags for 10s, 1s, 0.5s, and 0.1s, plus the `DTA` 1/100s counter.
- Halt wake now treats any pending SM530 gamma flag as a wake source, matching MAME's `m_gamma` behavior.
- The existing SM511/SM512 melody path is reused with SM530's `0xff` melody step mask.

## Known Limits

- Star Fox has an extra HMC sound-effect chip connected to SM530 `S` output edges in MAME. This branch does not emulate that separate sound chip; it only plays the SM530 melody/R output.
- MAME marks SM530 BP3/BP4 LCD counter coupling as TODO. This core currently follows the implemented MAME display-enable behavior and does not model those unimplemented BP effects.
- The CPU debug overlay uses SM530-specific rows for `K/KE` input and last RAM write address/data. Core debug rows 4-8 expose SM530 LCD output groups 0-3 and 11 so hardware snapshots can distinguish CPU execution failures from LCD normalization failures.
